# 国际化（i18n）改造设计文档

**日期**: 2026-06-22
**状态**: 已确认

---

## 背景

当前系统所有接口返回的 message 均为中文硬编码，集中在 `src/constant/messages.ts` 的 `MSG` 对象中。需要支持多语言，优先中英双语，后续可扩展。

## 目标

- 支持中英双语，通过 HTTP `Accept-Language` 请求头自动切换
- 规范化的 i18n 体系，后续新增语言只需添加翻译文件
- 彻底移除 `MSG` 硬编码常量，代码中直接使用翻译 key
- 支持消息中的动态插值（如 `{field} 已存在`）

## 技术选型

- **nestjs-i18n**（底层 i18next）— NestJS 官方推荐的 i18n 方案
- 翻译文件格式：**JSON**，按语言/模块拆分
- 语言解析：`AcceptLanguageResolver`（读 `Accept-Language` 头）

## Key 命名约定

- 全部小写、下划线分隔
- 调用形式：`t('user.not_found')`、`t('auth.login_required')`
- 带插值参数：`t('prisma.unique_constraint', { args: { field: '用户名' } })`

## 各层改造

### 1. I18nModule 注册

在 AppModule 中注册：

```ts
I18nModule.forRoot({
  fallbackLanguage: 'zh-CN',
  loaderOptions: {
    path: path.join(__dirname, '../i18n/'),
    watch: process.env.NODE_ENV === 'development',
  },
  resolvers: [AcceptLanguageResolver],
  typesOutputPath: path.join(__dirname, '../generated/i18n.generated.ts'),
})
```

- `fallbackLanguage: 'zh-CN'` — 不支持的请求语言兜底为中文
- `watch: true` — 开发时热更新翻译文件
- `typesOutputPath` — 生成翻译 key 类型，IDE 自动补全

### 2. BusinessException 重构

```ts
export class BusinessException extends HttpException {
  readonly businessCode: number;
  readonly i18nKey: string;
  readonly i18nArgs?: Record<string, string | number>;

  constructor(
    httpCode: number,
    i18nKey: string,
    options?: {
      businessCode?: number;
      args?: Record<string, string | number>;
    },
  ) {
    super(i18nKey, httpCode);
    this.i18nKey = i18nKey;
    this.businessCode = options?.businessCode ?? FAIL;
    this.i18nArgs = options?.args;
  }
}
```

调用示例：
```ts
// 无插值
throw new BusinessException(HttpStatus.BAD_REQUEST, 'user.not_found');

// 有插值
throw new BusinessException(HttpStatus.BAD_REQUEST, 'prisma.unique_constraint', {
  args: { field: '用户名' },
});

// 自定义业务码
throw new BusinessException(HttpStatus.UNAUTHORIZED, 'auth.login_required', {
  businessCode: UNAUTHORIZED,
});
```

### 3. HttpExceptionFilter — 统一翻译入口

- 从 `I18nContext.current()` 获取当前请求语言
- 对 `BusinessException`：用 `i18nKey` 翻译
- 对普通 `HttpException`：message 若为 key 格式则尝试翻译，否则直接返回
- Prisma 错误：从 `translatePrismaError` 改为 `resolvePrismaErrorKey`，返回 `{ key, args }`，走统一翻译

```ts
private handleHttpException(exception, request, response) {
  const httpCode = exception.getStatus();
  const key = exception instanceof BusinessException
    ? exception.i18nKey
    : exception.message;

  const i18n = I18nContext.current();
  const message = i18n.t(key, { args: exception.i18nArgs }) ?? key;

  response.status(httpCode).json({
    ...ApiResponse.fail(message),
    code: exception instanceof BusinessException ? exception.businessCode : FAIL,
  });
}
```

### 4. ApiResponse + ResponseInterceptor — 成功消息翻译

**ApiResponse.success()** 移除默认 message 参数（原 `MSG.COMMON.SUCCESS`），由 Interceptor 统一填充翻译后的消息：

```ts
// 改后：no default
static success<T>(data: T, message: string): ApiResponse<T> {
  return { code: SUCCESS, message, success: true, data };
}
```

**ResponseInterceptor** 通过 I18nContext 翻译：

```ts
const i18n = I18nContext.current();
return ApiResponse.success(data, i18n.t('common.success'));
```

Controller 手动调用 `ApiResponse.success()` 的场景（如 auth controller 的 login 返回）同样注入 I18nService 或者由 Interceptor 统一处理。

### 5. Service / Guard 改动

#### 5a. Guard + 异常抛出

所有 `MSG.X.Y` 引用改为翻译 key 字符串（见下表），语法从 `new BusinessException(httpCode, MSG.X.Y)` 变为 `new BusinessException(httpCode, 'x.y')`。

#### 5b. 返回 data.message 的翻译

以下 6 处 controller/service 返回的 `{ message: '...' }` 位于响应 `data` 层，不走 ExceptionFilter/Interceptor 翻译流程。这些 service 需要注入 `I18nService` 手动翻译：

| 文件 | 方法 | 翻译 key |
|------|------|----------|
| `user.service.ts` | `delUser` | `'user.delete_success'` |
| `user.service.ts` | `assignRoles` | `'user.assign_role_success'` |
| `role.service.ts` | `delRole` | `'role.delete_success'` |
| `role.service.ts` | `assignMenus` | `'role.assign_menu_success'` |
| `menu.service.ts` | `delMenu` | `'menu.delete_success'` |
| `auth.controller.ts` | `logout` | `'auth.logout_success'` |

代码示例：
```ts
// 改前
import { MSG } from '../constant';
return { message: MSG.USER.DELETE_SUCCESS };

// 改后
import { I18nService } from 'nestjs-i18n';
export class UserService {
  constructor(private i18n: I18nService) {}
  // ...
  return { message: this.i18n.t('user.delete_success') };
}
```

### 6. Prisma 错误翻译改造

`translatePrismaError` 方法拆分为 `resolvePrismaErrorKey`，返回 key + args：

```ts
private resolvePrismaErrorKey(error: Prisma.PrismaClientKnownRequestError) {
  const field = (error.meta?.target as string[])?.join('、') ?? '';
  switch (error.code) {
    case 'P2002': return { key: 'prisma.unique_constraint', args: { field } };
    case 'P2025': return { key: 'prisma.record_not_found' };
    case 'P2003': return { key: 'prisma.foreign_key_failed', args: { field } };
    case 'P2014': return { key: 'prisma.constraint_violation' };
    case 'P2021':
    case 'P2022': return { key: 'common.internal_error' };
    default:      return { key: 'prisma.unknown_error', args: { code: error.code } };
  }
}
```

## 影响评估

| 维度 | 说明 |
|------|------|
| 破坏性变更 | 是 — 响应 message 从中文变为多语言（前端需适配） |
| 数据库变更 | 无 |
| 新增依赖 | `nestjs-i18n` |
| 性能影响 | 翻译文件在启动时加载到内存，`t()` 为 O(1) 查表，无影响 |
